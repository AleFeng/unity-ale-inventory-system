# 装備システム（Equipment System）

- [説明ドキュメント](../README_JA.md) に戻る

装備システムは、プレイヤーがアイテムを「装備グループ」のスロットに装備して属性ボーナスを得られるようにします。装備グループは一式のスロット構造（複数の**スロットリスト**、各リストは複数の**装備スロット**を含む）を定義し、「アイテム制限」（機能タグ / 列挙制約）と「スロットフィルタ条件」で各スロットに装備できるアイテムを制約します。「装備属性フィールドリスト」は、どのアイテム属性を装備グループの合計属性ボーナスに集計するかを指定します。装備グループは設定カタログです。ランタイムの装備 / 解除 / 交換と属性ボーナス集計は `EquipmentRuntimeManager` が実行します。

# コアコンセプト

| 概念 | 説明 |
|------|------|
| グループタグ（`EquipmentGroupTag`） | **装備属性フィールドエントリ**をグループ化し、UI で合計属性ボーナスをグループ表示しやすくする（例：アイテムレベル / 主属性 / 副属性）。基本情報のみを保持し、属性フィールドは持たない |
| 装備グループテンプレート（`EquipmentGroupTemplate`） | 装備グループ作成のひな型。設定可能項目一式（スロットリスト + 装備属性フィールド）+ カスタム属性フィールド定義を保持し、分類フィルタにも使う |
| 装備グループ（`EquipmentGroup`） | 一式の装備のスロット構造：複数のスロットリスト（各リストは装備スロット + アイテム制限を含む）+ 装備属性フィールドリスト + カスタム属性値 |
| スロットリスト（`EquipmentSlotList`） | 一群の装備スロット + 統一された「アイテム制限」（機能タグ + 列挙制約） |
| 装備スロット（`EquipmentSlot`） | アイテムを 1 つ装備できるスロット。「スロットフィルタ条件」でさらに絞り込む |

# タブ構成

Inventory Editor 上部の「**装備システム**」タブをクリックします。クラフト / ショップシステムと対称な 3 カラムレイアウト：

```
左カラム：サブタブ【グループタグ / 装備グループテンプレート】+ 一覧 + 編集パネル
中央カラム：装備グループ一覧
右カラム：コンテキスト Inspector（グループタグ / 装備グループテンプレート / 装備グループ）
```

左カラムのサブタブで「グループタグ」または「装備グループテンプレート」を切り替えます。左カラムのエントリを選択すると右カラムにそのエントリの編集パネルが表示され、中央カラムの装備グループを選択すると右カラムに装備グループ Inspector が表示されます。

# グループタグ

基本情報のみを保持し、**属性フィールドは持ちません**。UI で装備グループの「装備属性フィールド」エントリをグループ表示するために使います。

| フィールド | 説明 |
|------|------|
| ID | 一意な識別子（装備属性フィールドエントリはこの ID で参照） |
| 名称 | `Text`（プレーンテキストのフォールバック + 任意のローカライズ参照。空のとき ID に退避） |
| 説明 | `Text`（プレーンテキストのフォールバック + 任意のローカライズ参照） |
| 色 | 一覧の色ドット |

# 装備グループテンプレート

**装備グループの設定可能項目一式の既定値を保持**（装備グループと同じ：スロットリスト + 装備属性フィールドリスト）し、さらにカスタム属性フィールド定義（スキーマ）を持ち、分類フィルタにも使います。

| フィールド | 説明 |
|------|------|
| 名称 / 色 | テンプレート名 + 一覧の色ドット |
| 装備倉庫 | [装備倉庫](#装備倉庫) を参照 |
| スロットリスト | [スロットリストとアイテム制限](#スロットリストとアイテム制限) を参照 |
| 装備属性フィールドリスト | [装備属性フィールドリスト](#装備属性フィールドリスト) を参照 |
| 整理ソート | ソート条件 + 整理優先度、下記参照 |
| カスタム属性フィールド | テンプレート定義の属性フィールド（装備グループはこれに基づいて属性値を調整） |

> 装備グループとテンプレートは `IEquipmentConfig`（装備倉庫 + スロットリスト + 装備属性フィールド + 整理ソート）を共有し、エディタは同じ描画（`EquipmentConfigDrawer`）を再利用します。
>
> **テンプレートから装備グループを作成するとき、テンプレートのすべての設定可能項目をディープコピー**（装備倉庫 + スロットリスト + 装備属性フィールド + 整理ソート）して初期データにします。以降、装備グループは**独立して編集可能**です（クラフトシステムの「テンプレートレベル読み取り専用」とは異なる）。
>
> **整理ソート**：テンプレートも装備グループも「整理ソート」（ソート条件 + 整理優先度）を設定でき、**装備選択パネル下部の「装備可能アイテム候補リスト」**（`UiwEquipmentCandidateList`）の表示順に使われます —— 候補リストにソートバーがあればプレイヤーが選んで昇順/降順を切り替えられ、なければ先頭条件が既定ソートになります。条件はテンプレートから装備グループへコピーされた後、独立して編集可能です（候補リストは装備グループ自身の設定を読む）。

# 装備グループ（中央カラム）

中央カラムは装備グループ一覧（選択した左カラムのテンプレートでフィルタ）です。「テンプレートから追加」はテンプレートから作成（設定をディープコピー + カスタム属性の既定値を調整）し、ID は自動生成されます。「クイック追加」は末尾を複製します。行をクリックして選択すると右カラムに装備グループ Inspector が表示され、「装備グループを削除」は右カラム上部にあります。

# 装備グループ Inspector（右カラム）

| フィールド | 説明 |
|------|------|
| ID | 一意な識別子（重複するとエクスポート検証でエラー） |
| 名称 / 説明 | `Text`（プレーンテキストのフォールバック + 任意のローカライズ参照。名称が空のとき ID / 基本説明に退避） |
| ソーステンプレート | 読み取り専用、カスタム属性フィールドを決定 |
| 装備倉庫 | 装備システム / 装備 UI が対話できる倉庫、[装備倉庫](#装備倉庫) を参照 |
| スロットリスト | ネスト編集（アイテム制限 + 装備スロット + スロットフィルタ条件）、下記参照 |
| 装備属性フィールドリスト | 合計属性ボーナスフィールド（属性フィールド ID + グループタグ）、下記参照 |
| カスタム属性値 | テンプレート定義由来のカスタム属性値 |

# 装備倉庫

「装備倉庫」（`equipmentInventoryRefs`、文字列の倉庫 ID リスト）は、装備システム / 装備 UI が**直接対話できる**プレイヤー倉庫を指定し、装備画面の**唯一の倉庫ソース**です（装備 UI は倉庫 ID を渡す必要がありません）。**順序が優先度**です：Inspector では各項目の前に番号が付き、左のドラッグハンドルで順序を調整できます（ショップの「取引倉庫」、クラフトの「クラフト倉庫」と `InventoryRefListDrawer` を共有）。

> **テンプレート / 装備グループの 2 段階 + ランタイムフォールバック**：テンプレートも装備グループも装備倉庫を設定でき、テンプレートから装備グループを作成すると、テンプレートの装備倉庫をディープコピーします。ランタイムでは「**有効な装備倉庫**」を取得します —— 装備グループ自身のリストが空でなければそれを使い、空のときは**自動的にソーステンプレートの装備倉庫にフォールバック**します（`EquipmentRuntimeManager.GetEquipmentInventories`）。したがって、テンプレート層だけで設定した場合や、このフィールド追加前に作成された装備グループ（自身のリストが空）でも、ランタイムで正常に動作します。

- **候補アイテムソース**：装備選択パネル下部の装備可能アイテムリストは、**すべての装備倉庫をまたいで**制限に合うアイテムを集計します。装備時はそのアイテムがある倉庫から取り出します。
- **装備の解除**：リストの **Index0 から順に試し**、最初に「入れられる」（空き ≥ 1）倉庫に入れます。どこにも入らなければ**解除しません**（false を返し、アイテムの喪失を回避）。
- **バックパックのブリッジ**：バックパックの右クリック自動装備では、装備倉庫内の倉庫由来のアイテムのみを扱います（装備倉庫が未設定ならソースを制限しません）。

# スロットリストとアイテム制限

1 つの装備グループは通常、複数のスロットリスト（例：「武器」「防具」「装飾品」）を含みます。各スロットリストエントリは既定で折りたたまれ、ID + 「詳細設定」の折りたたみのみ表示します。展開すると次を編集できます：

- **名称 / 説明**。
- **アイテム制限 → 機能タグ**：アイテムシステムで定義済みの機能タグから選び、このリストで装備できるアイテムを制限します。複数追加可能（左のドラッグハンドルで並べ替え）。
- **アイテム制限 → 列挙制約**：先に「列挙型」を選び、次に「列挙値」を複数選択します。複数追加可能（左のドラッグハンドルで並べ替え）。空の列挙値集合は「任意の値」を意味します。
- **装備スロット**：このリストのすべての装備スロット（下記参照）。

各スロットリストエントリの左にはドラッグハンドルがあり、スロットリスト全体の順序を並べ替えられます。

# 装備スロットとフィルタ条件

各スロットリストは複数の装備スロットを含みます。装備スロットの設定：

| フィールド | 説明 |
|------|------|
| ID | スロットの安定した識別子（ランタイムではこれで装備中アイテムを特定し、セーブする） |
| 名称 | UI 表示名（任意。空スロットのプレースホルダーにも使える） |
| スロットフィルタ条件 | スロットリストの制限に加え、このスロットに装備できるアイテムをさらに絞り込む |

**スロットフィルタ条件**（`EquipmentSlotFilter`）= 属性フィールド ID + 期待値：アイテムのその属性が期待値に等しい場合のみ装備可能（例：武器主タイプ = 近接片手、武器副タイプ = 剣）。期待値は汎用属性エディタでフィールド型に応じて編集します。同じスロットの複数のフィルタ条件は**すべて満たす**必要があります。これにより、職業 / キャラクタータイプごとに異なる装備制限を設定できます。

> 装備スロットは**ランタイムの装備中アイテムを保存しません** —— 装備中データは `EquipmentRuntimeManager` がスロット ID をキーに管理します。

# 装備属性フィールドリスト

「装備属性フィールドリスト」（`EquipmentAttributeDisplay`）は、アイテム上のどの属性フィールドを装備グループの**合計属性ボーナス**に集計するか（例：アイテムレベル / 攻撃力 / 防御力 / HP）を指定します。各エントリ：

| フィールド | 説明 |
|------|------|
| 属性フィールド ID | アイテムシステムの属性フィールドに関連付け（入力するか、ドロップダウンから定義済みフィールドを選択） |
| グループタグ | グループタグ ID を参照。UI のグループ表示用（例：主属性 / 副属性） |
| 表示名 | 任意の上書き（空のとき属性フィールド ID に退避） |

各エントリの左にはドラッグハンドルがあり、並べ替えできます。ランタイムではこのリストに従ってすべての装備中アイテムを合算します（[ランタイム API](#ランタイム-api) を参照）。

# 制限判定ルール

あるアイテムがあるスロットに装備できるかの判定は、**すべて AND** で行います：

1. **スロットリストの機能タグ**：アイテムは列挙されたすべての機能タグを**持つ**必要がある。
2. **スロットリストの列挙制約**：アイテムは**各**列挙制約を満たす必要がある —— その列挙型を参照する属性が存在し、その値が許可集合内にある。許可集合が空の場合は「任意の値」を意味する（それでもアイテムはその列挙型の属性を持つ必要がある）。
3. **スロットのフィルタ条件**：アイテムの対応する属性が**各**フィルタ条件の期待値に**等しい**必要がある。

3 つすべてを満たしたときにのみ、アイテムはそのスロットに装備できます。

# ランタイム API

`EquipmentRuntimeManager` は軽量シングルトン（初回アクセスで自動作成、`ShopRuntimeManager` に倣う）で、`装備グループ ID → (スロット ID → 装備中アイテム ID)` としてランタイム状態を管理します。装備グループのカタログは `InventoryDataManager` でクエリし、倉庫の読み書きは一律 `InventoryRuntimeManager` を通します（装備 / 解除はその `OnInventoryChanged` を発火し、バックパック UI はそれに従ってリフレッシュ）。

```csharp
using Ale.Inventory.Runtime;

var eq = EquipmentRuntimeManager.Instance;

// クエリ
string equipped = eq.GetEquipped("equip_player", "slot_weapon"); // そのスロットの装備中アイテム、空スロットは null を返す
bool occupied   = eq.IsSlotOccupied("equip_player", "slot_weapon");

// 制限マッチング
var group    = InventoryDataManager.Instance.GetEquipmentGroup("equip_player");
var slotList = group.slotLists[0];
bool canList = eq.ItemMatchesSlotList(slotList, "sword_01");                 // スロットリストの制限を満たす？
bool canSlot = eq.ItemMatchesSlot(slotList, slotList.slots[0], "sword_01");  // さらにそのスロットのフィルタを満たす？

// スロットの自動検索 + 自動装備（バックパックから 1 つ取り出し、最初に装備できる空スロットに入れる）
if (eq.TryFindEquipSlot("equip_player", "sword_01", out var slId, out var slotId)) { /* ... */ }
bool autoOk = eq.TryAutoEquip("equip_player", "sword_01", "バックパック");   // 空スロットのみ埋める
// クイック装備（置き換え可）：① 優先スロット preferredSlotId（占有中なら置き換え）→ ② 空スロット → ③ 制限を満たす最初の占有済みスロット（Index0 から、元のアイテムをソース倉庫へ戻す）
// UI の右クリッククイック装備はこの経路。装備選択パネルが開いているときは、現在選択中のスロットを優先スロットとして渡す
bool quickOk = eq.TryAutoEquipOrReplace("equip_player", "sword_01", "バックパック", preferredSlotId: "slot_weapon");

// 指定スロットで装備 / 解除 / 交換（倉庫と連携して搬送。占有スロットに装備するとき、旧アイテムをソース倉庫へ戻す）
bool eOk = eq.Equip("equip_player", "weapon_list", "slot_weapon", "sword_01", "バックパック");
bool uOk = eq.Unequip("equip_player", "slot_weapon", "バックパック");      // 指定倉庫へ解除（入らなければ失敗、解除しない）
// 装備グループ設定の「装備倉庫」へ解除：Index0 から最初に入れられる倉庫を探す。未設定のときは fallback（ここではバックパック）に退避
bool ucOk = eq.UnequipToConfigured("equip_player", "slot_weapon", "バックパック");
string inv = eq.FindEquipmentInventoryFor("equip_player", "sword_01"); // このアイテムを入れられる装備倉庫のうち最初の倉庫 ID（なければ null）
bool sOk = eq.SwapSlots("equip_player", "slot_weapon", "slot_offhand"); // 同一グループのスロット↔スロット交換（双方が相手の制限を満たす必要あり）

// 合計属性ボーナス（「装備属性フィールドリスト」に従いすべての装備中アイテムを集計、グループタグを付けて UI グループ化に使う）
foreach (var b in eq.GetTotalBonuses("equip_player"))
    Debug.Log($"{b.Label}({b.GroupTag}) = {b.Total}");   // EnumIntPair のときは列挙 Key ごとに複数エントリに分割、b.EnumValue が列挙値

// イベント + セーブ
eq.OnEquipmentChanged += groupId => { /* 装備 UI をリフレッシュ */ };
var save = eq.GetSaveData();          // ゲーム層の SaveManager がシリアライズ
eq.LoadSaveData(save);                // 読み込みで復元
eq.ResetAll();                        // クリア（新規ゲーム開始など）
```

- **装備**：`Equip` はまず制限を検証し、ソース倉庫から 1 つ取り出します。スロットが占有済みなら旧アイテムをソース倉庫へ戻します（先に取ってから入れるので、正味の占有は不変。戻せなければロールバックし、アイテムを失わない）。ソース倉庫が未指定の場合はスロットを設定するだけです（旧アイテムは直接置き換わり、呼び出し側の自己責任）。
- **解除**：`Unequip` はアイテムを指定した対象倉庫へ戻します。入らなければ false を返し、解除しません。UI の右クリック解除は `UnequipToConfigured(groupId, slotId)` を通ります —— 装備グループの「装備倉庫」リストの Index0 から最初に入れられる倉庫を探し、どこにも入らない（または未設定でフォールバック倉庫もない）場合は解除せず、アイテムを破棄しません。
- **属性ボーナス**：`GetTotalBonuses` は各装備属性フィールドについて、すべての装備中アイテムをまたいで集計し、**記録方式はソース属性の `AttributeValue.Type` によって異なります**：
  - **`EnumIntPair`**：列挙 Key ごとに分割 —— 各列挙 Key が単独の `EquipmentBonus` になり、その整数値が `Total` に累積されます。`EnumTypeRef`/`EnumValue` がソース列挙を記録し、表示名は装備属性フィールドの「表示名ソース（列挙フィールド）」（`EquipmentAttributeDisplay.enumLabelAttrId`）を通じて対応する列挙項目の String / Text 属性から解決され、未設定なら列挙項目名にフォールバックします。典型的には「キャラクター属性値ボーナス」（例：筋力 +13、敏捷 +5）に使います。**実際の列挙項目に解決できない Key（列挙型の欠落 / 列挙項目の削除）は表示されません。**
  - **`StringIntPair`**：文字列 Key ごとに分割累積し、表示名はその文字列 Key です。**空文字列 Key は表示されません。**
  - **その他の配列型**：要素インデックスごとに分割 —— 各インデックス位置に 1 エントリ、同じインデックス位置で各アイテムを累積。
  - **スカラー型**：1 エントリに集約し、`AttributeValue.ToComparableNumber()` で合算（数値は直接取得、ベクトルは大きさ）。[属性システム - ソート比較数値](AttributeSystem_JA.md) を参照。

# 装備 UI

装備 UI は `Runtime/UI/`（アセンブリ `Ale.Inventory.UI`）にあります：

| コンポーネント | 説明 |
|------|------|

| `UiwEquipmentSlot` | 装備スロット：`UiwInventoryItemSlotBase` を継承。装備中アイテムを表示。左 / 右クリックイベント。ドラッグソース（ドラッグアウトで交換）+ ドロップ対象（装備 / 交換）+ 緑/赤の有効性オーバーレイ |
| `UiwEquipmentBonusPanel` / `UiwEquipmentBonusEntry` | 属性ボーナスパネル：グループタグごとに合計属性ボーナスを表示。ボーナスが 1 つもないときは空状態ヒントを表示（`emptyText`、ローカライズテーブル/エントリ参照 `emptyTextLocalized*` に対応） |
| `UiwEquipmentSelectPanel` | 装備選択パネル：上部のスロットグループ切り替えバー（左右 + N/M）+ 中央の現在のスロットリスト + 下部の装備可能アイテムリスト + 退出（ボタン / 空白部の右クリック） |

| `GridCellDragHandler` | アイテムセルのドラッグ中継コンポーネント（`UiwInventoryItemCell` またはその子に付与、`Runtime/UI/ItemList/` にある）：**グリッドリストに接続済み**（バックパック）のときは `UiwInventoryItemGridList` へ転送し、ドラッグ終了時に落下点で判定（装備スロット→装備、アイテムセル→入れ替え）。**グリッドリストに未接続**（候補リスト）のときは「装備スロットへドラッグして装備」を駆動（`UiwEquipmentDragContext` 経由、ゴースト + 緑/赤の有効性付き）。右クリッククイック装備は本コンポーネントでは処理しない —— `UiwInventoryItemCell` が「アイテム右クリック」をブロードキャストし、`UiwEquipmentView` が購読して統一処理するので、セルはどのコンテナでも一貫して対話できる |
| `UiwEquipmentDragContext` | 装備ドラッグのグローバルコンテキスト（ペイロード + カーソル追従のゴースト + ソースアイコンの復帰） |

## 対話

- **選択パネルで装備**：装備スロットを左クリック → 選択パネルを開く → スロットグループを切り替え → 下部の候補アイテムを**右クリックでクイック装備**（`UiwEquipmentView` が「アイテム右クリック」イベントを購読、装備優先度：**① 選択パネルで現在選択中の装備スロット**（占有中なら置き換え）→ ② 最初に装備できる空スロット → ③ 制限を満たす最初の占有済みスロット（Index0 から、元のアイテムをソース倉庫へ戻す））、または**左ドラッグで装備スロットへ**装備。候補アイテムを左シングルクリックしても装備は発火しない。候補アイテムとバックパックアイテムは**一貫して対話**（右クリックでクイック装備 / 左ドラッグで装備スロットへ装備）。
- **解除**：装備スロットを右クリック → 解除。装備グループの「装備倉庫」リストの Index0 から最初に入れられる倉庫へ入れる。どこにも入らなければ解除しない（アイテムを破棄しない）。
- **装備スロット同士の交換**：ある装備スロットを別の装備スロット（同一グループ）へドラッグ。双方が相手の制限を満たすとき交換。
- **装備スロットをバックパックセルへドラッグ**（そのバックパックの `dragSort = true` が必要）：装備スロットをバックパックのあるセルへドラッグ —— **空セル**に落とす → 解除してそのセルに正確に入れる。**アイテムのあるセル**に落とす → そのアイテムがソース装備スロットに装備できるなら**位置を交換**（そのセルのアイテムを装備スロットへ、元の装備アイテムをそのセルへ戻す）。装備できなければキャンセルし、変更しない。
- **ドラッグの有効性**：装備スロット上にドラッグホバー中、装備できるかで緑（可）/ 赤（不可）を表示。
- **バックパックとの相互運用**：バックパック画面でアイテムを**右クリック** → クイック装備（`UiwEquipmentView` は開いているとき「アイテム右クリック」イベントを自動購読、追加の配線不要。空スロットがないときは制限を満たす最初の占有済みスロットを置き換え）。バックパックの**グリッド**セルは装備スロットへ**ドラッグ**して装備可能（そのバックパックの `dragSort = true` が必要。順序リストはドラッグ不可、右クリックを使う）。

プレハブ作成と共通コンポーネントは [UI コンポーネントガイド](UIComponentGuide_JA.md) を参照してください。

> ヒント：装備スロットのプレハブに、既定で無効の「有効性オーバーレイ画像」（`validityOverlay` に接続）を付けて初めて緑/赤のヒントが表示されます。選択パネルのルートノードには「空白部で右クリック退出」のために raycast グラフィックが必要です。ドラッグにはシーン内に EventSystem が必要です。
