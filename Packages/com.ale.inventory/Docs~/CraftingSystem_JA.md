クラフトシステムは「ブループリント（レシピ）」で「材料を消費 → アイテムを産出」を記述します。ブループリントはアイテムシステムのアイテム ID を参照し、クラフト倉庫から材料を差し引き、クラフト倉庫へ産出物を入れます。ブループリントは設定カタログです。ランタイムのクラフト動作は `CraftingRuntimeManager` が実行し、連続クラフトの回数 / 計時 / 進捗は UI 層が駆動します。

# コアコンセプト

| 概念 | 説明 |
|------|------|
| グループタグ（`CraftingGroupTag`） | ブループリントをグループ化し UI フィルタを容易にする。各ブループリントは主グループ 1 + 副グループ複数 |
| ブループリントテンプレート（`CraftingBlueprintTemplate`） | カスタム属性フィールド + 設定の既定値 + テンプレートレベルの整理ソート。ブループリント作成のひな型であり、分類（防具 / 武器 / 食品 …）にも使う |
| ブループリント（`CraftingBlueprint`） | 1 つのレシピ：産出 / 消費アイテムのリスト + 作成時間 / 回数 + クラフト倉庫 + UI 設定 |
| クラフト倉庫 | 順序付き倉庫 ID リスト：優先度順に材料ソースと産出先として使う |

# タブ構成

Inventory Editor 上部の「**クラフトシステム**」タブをクリックします。ショップシステムと対称な 3 カラムレイアウト：

```
左カラム：サブタブ【グループタグ / ブループリントテンプレート】+ 一覧 + 編集パネル
中央カラム：ブループリント一覧
右カラム：コンテキスト Inspector（グループタグ / ブループリントテンプレート / ブループリント）
```

左カラムのサブタブで「グループタグ」または「ブループリントテンプレート」を切り替えます。左カラムのエントリを選択すると右カラムにそのエントリの編集パネルが表示され、中央カラムのブループリントを選択すると右カラムにブループリント Inspector が表示されます。

# グループタグ

基本情報のみを保持し、**カスタム属性フィールドは持ちません**。UI でブループリントをグループ化してフィルタするために使います。

| フィールド | 説明 |
|------|------|
| ID | 一意な識別子（ブループリントはこの ID で参照） |
| 名称 | `Text`（プレーンテキストのフォールバック + 任意のローカライズ参照。空のとき ID に退避） |
| 説明 | `Text`（プレーンテキストのフォールバック + 任意のローカライズ参照） |
| 色 | 一覧の色ドット |

# ブループリントテンプレート

カスタム属性フィールド + ブループリントの設定可能項目一式の既定値（ブループリント作成時にコピー）を定義し、**テンプレートレベルの整理ソート**（このテンプレート配下の全ブループリントの UI 一覧での並び順。ブループリント自身は別途ソートを設定しない）を保持します。

| フィールド | 説明 |
|------|------|
| 名称 / 色 | テンプレート名 + 一覧の色ドット |
| 作成時間 | 1 回作成するのに必要な秒数 |
| 連続作成回数 | [作成可能回数と連続作成回数](#作成可能回数と連続作成回数) を参照 |
| クラフト倉庫 | 倉庫 ID のリスト（順序付き） |
| 数字フォーマット | 参照する数字フォーマット設定の名称 |
| 整理リスト / 整理優先度 | テンプレートレベルのソート条件（主 + 副） |
| 属性フィールド表示 | UI に表示する属性（Label + 属性フィールド ID） |
| 属性フィールドリスト | テンプレートのカスタム属性フィールド |

> ブループリントとブループリントテンプレートは `ICraftingConfig` を共有し、設定項目は同一で、エディタは同じ描画を再利用します。

# ブループリント（中央カラム）

中央カラムはブループリント一覧（選択した左カラムのテンプレートでフィルタ）です。新規ブループリントはテンプレートから作成され、ID は自動生成されます。行をクリックして選択すると右カラムにブループリント Inspector が表示され、「ブループリントを削除」は右カラム上部にあります。

# ブループリント Inspector（右カラム）

| フィールド | 説明 |
|------|------|
| ID | 一意な識別子（重複するとエクスポート検証でエラー） |
| 名称 / 説明 | `Text`（プレーンテキストのフォールバック + 任意のローカライズ参照。名称が空のとき ID / 基本説明に退避） |
| ソーステンプレート | 設定可能項目とカスタム属性を決定 |
| 主グループタグ | 単一選択、グループタグ ID を参照 |
| 副グループタグ | 複数選択 |
| 産出アイテムリスト | Index0 = 主産出（UI 表示）、それ以外は副産出 |
| 消費アイテムリスト | 1 回作成するのに必要な材料 |
| 作成パラメータ（作成時間 / 連続作成回数） | **ブループリントレベル、編集可能** |
| クラフト倉庫 / UI 設定（数字フォーマット + 属性フィールド表示） | **テンプレートレベル、ブループリントエントリでは読み取り専用表示**。ソーステンプレートをミラーし、「ブループリントテンプレート」でのみ変更可能 |
| 属性フィールド値 | テンプレート定義由来のカスタム属性値 |

> **テンプレートレベルの設定**：クラフト倉庫と UI 設定（数字フォーマット + 属性フィールド表示）は「ブループリントテンプレート」でのみ設定できます。ブループリントエントリは常にソーステンプレートのこれらのフィールドをミラーし（`CraftingBlueprint.RebuildAttributes` が同期）、ブループリント Inspector では読み取り専用で表示され、個別には変更できません。ソーステンプレートのないブループリントは、これらの項目を設定する前にまずテンプレートを指定する必要があります。

# レシピ：産出と消費

産出（`outputs`）も消費（`inputs`）も「アイテム + 数量」（`CraftingItemAmount`：`itemId` + `count`）のリストです。

- **産出 Index0 = 主産出**：UI 詳細のアイコン / 名称 / 説明は主産出アイテムから取得。それ以外は副産出（詳細では小アイコンで表示、ホバーで詳細をポップ）。
- **消費リスト**：1 回作成するのに必要な材料。ランタイムでは要求に対して保有量を項目ごとに検証。
- ブループリントが存在しないアイテムを参照すると、エクスポート検証でエラーになります。

# クラフト倉庫

`craftInventoryRefs` は順序付き倉庫 ID リストです（**テンプレートレベルの設定：「ブループリントテンプレート」で編集、ブループリントエントリは読み取り専用で継承**）：

- **材料ソース**：材料を差し引くとき、リスト順（優先度）で各倉庫から累計して差し引く。
- **産出先**：産出物を入れるとき、リスト順で各倉庫の残り容量に分配。すべて入り切らない場合、超過分は容量に応じて破棄（ショップ取引と同じ）。
- **保有量集計**：作成可能回数 / 保有量はすべてのクラフト倉庫をまたいで集計。

# 属性フィールド表示

「属性フィールド表示」（`CraftingAttributeDisplay`：`label` + `attrId`）は、ブループリントエントリ / 詳細に**主産出アイテム**のどの属性を「Label 値」の形（例：「レベル 5」「価値 120」）で表示するかを制御します。**テンプレートレベルの設定：「ブループリントテンプレート」で編集（左のハンドルをドラッグして並べ替え可能）、ブループリントエントリは読み取り専用で継承。**

値は `AttributeValue.ToDisplayString()` がフィールド型に応じて組み立てます（数値は直接変換、ベクトルは全成分、StringIntPair は `key: value` 表示など）。[属性システム - 表示文字列](AttributeSystem_JA.md#表示文字列todisplaystring) を参照してください。

# 作成可能回数と連続作成回数

混同しやすい 2 つの概念：

| 概念 | 意味 | 計算 |
|------|------|------|
| **作成可能回数（材料）** | 現在の材料で何回作れるか | 各消費アイテムの `floor(保有量 / 1 回の消費)` の最小値 |
| **連続作成回数（`maxCraftCount`）** | 1 回の「作成」動作で連発を許す一括上限 | `1` = 1 回のみ。`-1` = 無制限 |

- UI 詳細の「作成可能：N」が表示するのは**材料の作成可能回数**（`GetMaxCraftableByMaterials`）です。
- 作成回数セレクター（連続作成の一括）の上限 = 材料の作成可能回数 と `maxCraftCount` の小さい方（`GetMaxCraftable`）。

# ランタイム API

`CraftingRuntimeManager` は軽量シングルトンで、自身の状態を持たず、セーブもしません。アイテムデータは `InventoryDataManager` でクエリし、倉庫の読み書きは `InventoryRuntimeManager` を通します（消費 / 産出はその `OnInventoryChanged` を発火し、UI はそれに従ってリフレッシュ）。

```csharp
using Ale.Inventory.Runtime;

var cm = CraftingRuntimeManager.Instance;
CraftingBlueprint bp = InventoryDataManager.Instance.GetCraftingBlueprint("craft_sword");

int owned = cm.GetOwnedAcross(bp, "iron_ingot");          // クラフト倉庫をまたいだ保有量
int byMat = cm.GetMaxCraftableByMaterials(bp);            // 材料の作成可能回数（表示用）
int max   = cm.GetMaxCraftable(bp);                       // セレクター上限（連続作成回数に制約される）
bool can  = cm.CanCraftOnce(bp);                          // 材料が 1 回分足りるか

// 1 回作成を実行（先に全消費が十分か検証し、次に倉庫をまたいで材料を差し引き、産出を入れる）
bool ok = cm.CraftOnce(bp);          // または cm.CraftOnce("craft_sword")
```

連続クラフト = UI 層が `CraftOnce` をループで呼び、計時 / 進捗を駆動します（`UiwCraftingDetail` を参照）。

# クラフト UI

クラフト UI は `Runtime/UI/View/Crafting/` にあり、構造はバックパックと対称です：

| コンポーネント | 説明 |
|------|------|
| `UiwCraftingView` | クラフトメイン画面：ブループリントテンプレートタブ + 名称検索 + グループ折りたたみタブ + ソート整理バー + ブループリント仮想リスト + ブループリント詳細 |
| `UiwCraftingDetail` | ブループリント詳細：主 / 副産出、消費リスト、作成可能回数、作成回数選択（数値カウンター）、作成 / 停止 + プログレスバー |
| `UiwCraftingBlueprintCell` | ブループリント一覧エントリ |
| `UiwCraftingInputCell` | 消費アイテム行（アイコン / 名称 / 要求 / 保有） |
| `UiwCraftingBlueprintList` | ブループリント仮想スクロールリスト |
| `UiwCraftingGroupFilter` | グループ折りたたみタブ（主グループから副グループを展開でき、`UiwFoldTab` を再利用） |

フィルタパイプライン：テンプレート → グループ（主 / 副）→ 名称検索。その後、選択したテンプレートの整理設定でソート。プレハブ作成とパラメータは [UI コンポーネントガイド](UIComponentGuide_JA.md) を参照してください。
