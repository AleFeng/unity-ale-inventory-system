# 打开编辑器

1. **创建数据文件**：`Project 面板右键 > Create > Inventory System > Inventory Database`（或欢迎窗口「创建新数据文件」）。
2. **打开编辑器**：选中 `.asset` 在 Inspector 点「在 Inventory Editor 中编辑」，或菜单 `Tools > Inventory System > Inventory Editor`。
3. 顶部点击「**道具系统**」页签。

编辑器为三列布局：**左列**（定义配置）| **中列**（道具列表）| **右列**（道具 Inspector）。

# 页签结构

道具系统页签的左列顶部有三个子页签：**枚举类型**、**功能标签**、**道具模板**。三者都是「定义」，供道具引用与复用。

```
左列：枚举类型 / 功能标签 / 道具模板（子页签 + 列表 + 编辑面板）
中列：道具列表（模板过滤标签 + 搜索 + 拖拽重排）
右列：选中道具的 Inspector
```

# 枚举类型

用于定义需要有限选项的字段，例如品质、职业、部位、元素属性。

- **新建**：左列底部「新建枚举」按钮。
- **编辑**（点击枚举名，右列出现编辑面板）：

| 字段 | 说明 |
|------|------|
| 名称 | 枚举类型唯一名称，属性字段通过此名称引用 |
| 颜色 | 列表中的色点标记 |
| 枚举项列表 | 每项含显示名 + 只读枚举值（整数）；**拖拽 ≡ 句柄**调整显示顺序（不改变已存储的值）；✕ 删除 |
| 枚举项属性字段 | 每个枚举项可附加一组自定义属性字段（如「装备」枚举项携带属性加成），配置同功能标签 |
| 「添加枚举项」 | 系统自动分配不重复枚举值，删除后**永不复用** |

> 枚举值由系统分配且只读，不受显示顺序影响。详见 [属性系统 - 枚举类型与稳定引用](AttributeSystem.md#枚举类型与稳定引用)。

# 功能标签

给道具附加一组能力标记，同时为该标记携带的属性字段提供定义。例如「装备」标签携带「部位」「攻击力」等字段。

- **新建**：「新建功能标签」按钮。
- **编辑**：

| 字段 | 说明 |
|------|------|
| 名称 | 标签唯一名称；道具通过此名称引用 |
| 说明 | 可选描述（仅编辑器显示） |
| 属性字段列表 | 该标签携带的属性字段定义；拖拽调整顺序；✕ 删除 |
| 「添加字段」 | 添加属性字段（类型见 [属性字段类型参考](AttributeSystem.md#属性字段类型参考)） |

**属性字段配置项**：ID（唯一 Key，支持中文）、类型、是否数组、枚举类型（仅 Enum）、默认值。

# 道具模板

道具模板定义一组**基础属性字段**，所有从该模板创建的道具都会携带。

- **新建**：「新建道具模板」按钮。
- **编辑**：

| 字段 | 说明 |
|------|------|
| 名称 | 模板唯一名称 |
| 颜色 | 道具列表中该模板道具的色点 |
| 功能标签 | 可将功能标签**锁定**到模板（该模板的道具自动携带这些标签及字段，不可在道具层取消） |
| 属性字段列表 | 模板自有属性字段 |

> **设计建议**：通用字段（名称、描述、图标、品质）放模板属性；类别专属字段（攻击力、恢复量）放功能标签属性，再用模板锁定对应标签。

# 道具列表（中间列）

```
[ 全部 ][ 武器 ][ 防具 ][ 消耗品 ]   ← 模板过滤标签栏（单选）
[ 🔍 搜索框 ][ 从模板添加 ▾ ][ 快速添加 ]
─────────────────────────────────────
≡ ● 模板名 | ID | 属性1 | 属性2 …  ✕
```

| 操作 | 说明 |
|------|------|
| 模板过滤标签栏 | 「全部」显示所有道具；点击模板名只显示该模板道具 |
| 搜索框 | 按道具 ID / 属性字符串值 / 枚举显示名 / 标签名模糊过滤 |
| 从模板添加 | 选模板生成新道具（ID 自动 `item_N`） |
| 快速添加 | 克隆列表最后一个道具 |
| 拖拽 ≡ 句柄 | 在当前过滤结果内调整道具顺序 |
| 点击行 / ✕ | 选中 / 删除 |

**重复 ID 检测**：ID 为空或重复的行以红色高亮，底部状态栏显示警告，非空重复时导出按钮禁用。

# 道具 Inspector（右侧列）

选中道具后右列显示完整编辑界面：

- **ID**：输入框；重复或空白时背景变红。
- **来源模板**：只读，创建后不可改。
- **功能标签**：
  - 模板锁定的标签：灰色只读，标注「由模板锁定」。
  - 道具自选标签：Toggle 列表；勾选自动添加该标签字段，取消自动移除。
- **属性（分组展示，可折叠）**：

| 组 | 来源 |
|----|------|
| 模板：XXX | 模板自有属性字段 |
| 标签名（模板锁定） | 模板锁定标签的属性字段 |
| 标签名 | 道具自选标签的属性字段 |
| 其他（来源已删除） | 来源模板 / 标签已删除的历史字段（下次 Rebuild 时清除） |

同名字段来自多个来源时显示冲突警告，按优先级（模板自有 → 模板锁定标签 → 自选标签）先到先得。枚举字段选中某值后会在其下方只读展开该枚举项的子属性。

# 仓库属性：重量与堆叠

道具 Inspector 中还有两个供仓库系统使用的字段：

| 字段 | 说明 |
|------|------|
| 重量 | 浮点；0 = 无重量（不参与仓库重量上限计算） |
| 堆叠上限 | 整数；0 = 无限堆叠，1 = 不可堆叠，>1 = 具体上限 |

# 运行时查询

道具数据为静态定义，运行时经 `InventoryDataManager` 查询：

```csharp
using InventorySystem.Runtime;

// 按 ID 取道具
Item item = InventoryDataManager.Instance.GetItem("sword_01");

// 取属性值（强类型 + fallback）
int   atk  = item.GetAttributeValue<int>("攻击力", 0);
string nm  = item.GetAttributeValue<string>("名称");

// 取枚举显示名
EnumType quality = InventoryDataManager.Instance.GetEnumType("品质");
string qName = quality?.GetItemByValue(item.GetAttributeValue<int>("品质"))?.name;

// 判断标签
bool isEquip = InventoryDataManager.Instance.ItemHasTag("sword_01", "装备");
```

属性读取的完整 API、显示字符串与排序折算见 [属性系统](AttributeSystem.md)。

# 道具相关 UI 组件

`Runtime/UI/Item/` 下的格子组件按道具静态数据自动取图标 / 名称 / 品质等显示：

| 组件 | 说明 |
|------|------|
| `UiwInventoryItemSimple` | 简版格子：仅图标 + 数量（货币栏等） |
| `UiwInventoryItemDetail` | 完整格子：图标 / 名称 / 描述 / 品质背景 / 数量 / 价格 / 悬停高亮 / 堆叠已满动画 |
| `UiwShopItemDetail` | 商店格子：品质背景 + 图标 + 名称 / 单价 + 剩余可交易次数 + 数量选择 |
| `UiwCraftingBlueprintCell` | 蓝图条目：主产出图标 + 蓝图名 + 按蓝图配置显示主产出道具属性 |

各组件的 Inspector 参数与预制体制作见 [UI 组件指南](UIComponentGuide.md)。

# 常见问题

**Q：修改枚举项显示顺序，已有道具的枚举值会变吗？**
A：不会。枚举属性存的是不可变的整数枚举值，与显示顺序无关。

**Q：删除功能标签后道具的字段数据丢失吗？**
A：调用 `RebuildAttributes` 后来源已删除的字段会被移除；移除前会在 Inspector「其他（来源已删除）」分组中可见。

**Q：Inspector 显示「未找到枚举类型」？**
A：检查枚举类型名是否被改名；在属性字段定义中重新选择枚举类型即可修复。

**Q：导出按钮灰色？**
A：存在非空重复 ID 的道具 / 仓库 / 商店 / 蓝图，先修复红色高亮条目（空白 ID 会在导出时自动跳过，不阻塞）。
