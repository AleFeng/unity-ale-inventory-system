# アイテムシステム（Item System）

- [説明ドキュメント](../README_JA.md) に戻る

アイテムシステムはプラグイン全体の基盤です：ゲーム内のすべてのアイテムとその属性を定義します。倉庫 / ショップ / クラフトシステムはいずれもアイテムシステムのアイテム ID を参照します。本ドキュメントでは、Inventory Editor の「アイテムシステム」タブで列挙型、機能タグ、アイテムテンプレート、アイテムを設定する方法と、ランタイムでアイテムデータをクエリする方法を説明します。

# エディタを開く

1. **データファイルを作成**：`Project パネルで右クリック > Create > Inventory System > Inventory Database`（またはウェルカムウィンドウの「新規データファイルを作成」）。
2. **エディタを開く**：`.asset` を選択して Inspector で「Inventory Editor で編集」をクリック、またはメニューの `Tools > Inventory System > Inventory Editor`。
3. 上部の「**アイテムシステム**」タブをクリック。

エディタは 3 カラムレイアウトです：**左カラム**（定義設定）| **中央カラム**（アイテム一覧）| **右カラム**（アイテム Inspector）。

# タブ構成

アイテムシステムタブの左カラム上部には 3 つのサブタブがあります：**列挙型**、**機能タグ**、**アイテムテンプレート**。3 つとも「定義」であり、アイテムが参照・再利用します。

```
左カラム：列挙型 / 機能タグ / アイテムテンプレート（サブタブ + 一覧 + 編集パネル）
中央カラム：アイテム一覧（テンプレートフィルタタグ + 検索 + ドラッグ並べ替え）
右カラム：選択したアイテムの Inspector
```

# 列挙型

品質、職業、部位、属性など、有限の選択肢を持つフィールドを定義するために使います。

- **新規**：左カラム下部の「列挙を新規作成」ボタン。
- **編集**（列挙名をクリックすると右カラムに編集パネルが表示）：

| フィールド | 説明 |
|------|------|
| 名称 | 列挙型の一意な名称。属性フィールドはこの名称で参照します |
| 色 | 一覧内の色ドットマーカー |
| 列挙項目リスト | 各項目は表示名 + 読み取り専用の列挙値（整数）を持ちます。**ドラッグ ≡ ハンドル**で表示順序を調整（保存済みの値は変わりません）。✕ で削除 |
| 列挙項目の属性フィールド | 各列挙項目に一組のカスタム属性フィールドを付加できます（例：「装備」列挙項目が属性ボーナスを持つ）。設定方法は機能タグと同じ |
| 「列挙項目を追加」 | システムが重複しない列挙値を自動採番し、削除後も**決して再利用しません** |

> 列挙値はシステムが割り当て、読み取り専用で、表示順序の影響を受けません。詳細は [属性システム - 列挙型と安定した参照](AttributeSystem_JA.md#列挙型と安定した参照) を参照してください。

# 機能タグ

アイテムに一組の能力マーカーを付加し、同時にそのマーカーが持つ属性フィールドの定義を提供します。例えば「装備」タグは「部位」「攻撃力」などのフィールドを持ちます。

- **新規**：「機能タグを新規作成」ボタン。
- **編集**：

| フィールド | 説明 |
|------|------|
| 名称 | タグの一意な名称。アイテムはこの名称で参照します |
| 説明 | 任意の説明（エディタ表示のみ） |
| 属性フィールドリスト | このタグが持つ属性フィールドの定義。ドラッグで順序変更。✕ で削除 |
| 「フィールドを追加」 | 属性フィールドを追加（型は [属性フィールド型リファレンス](AttributeSystem_JA.md#属性フィールド型リファレンス) を参照） |

**属性フィールドの設定項目**：ID（一意なキー、非 ASCII 文字対応）、型、配列かどうか、列挙型（Enum のみ）、既定値。

# アイテムテンプレート

アイテムテンプレートは一組の**基本属性フィールド**を定義し、そのテンプレートから作成されたすべてのアイテムがそれらを持ちます。

- **新規**：「アイテムテンプレートを新規作成」ボタン。
- **編集**：

| フィールド | 説明 |
|------|------|
| 名称 | テンプレートの一意な名称 |
| 色 | アイテム一覧でこのテンプレートのアイテムに付く色ドット |
| 機能タグ | 機能タグをテンプレートに**固定**できます（このテンプレートのアイテムは自動的にこれらのタグとフィールドを持ち、アイテム側で解除できません） |
| 属性フィールドリスト | テンプレート固有の属性フィールド |

> **設計のヒント**：共通フィールド（名称、説明、アイコン、品質）はテンプレート属性に、カテゴリ固有のフィールド（攻撃力、回復量）は機能タグ属性に置き、対応するタグをテンプレートに固定します。

# アイテム一覧（中央カラム）

```
[ すべて ][ 武器 ][ 防具 ][ 消耗品 ]   ← テンプレートフィルタタグバー（単一選択）
[ 🔍 検索ボックス ][ テンプレートから追加 ▾ ][ クイック追加 ]
─────────────────────────────────────
≡ ● テンプレート名 | ID | 属性1 | 属性2 …  ✕
```

| 操作 | 説明 |
|------|------|
| テンプレートフィルタタグバー | 「すべて」はすべてのアイテムを表示。テンプレート名をクリックするとそのテンプレートのアイテムのみ表示 |
| 検索ボックス | アイテム ID / 属性の文字列値 / 列挙表示名 / タグ名であいまいフィルタ |
| テンプレートから追加 | テンプレートを選んで新しいアイテムを生成（ID は自動で `item_N`） |
| クイック追加 | 一覧の最後のアイテムを複製 |
| ドラッグ ≡ ハンドル | 現在のフィルタ結果内でアイテムの順序を調整 |
| 行をクリック / ✕ | 選択 / 削除 |

**ID 重複検出**：ID が空または重複している行は赤くハイライトされ、下部のステータスバーに警告が表示され、空でない重複がある間はエクスポートボタンが無効になります。

# アイテム Inspector（右カラム）

アイテムを選択すると、右カラムに完全な編集 UI が表示されます：

- **ID**：入力ボックス。重複または空白のとき背景が赤くなります。
- **ソーステンプレート**：読み取り専用、作成後は変更不可。
- **機能タグ**：
  - テンプレート固定のタグ：グレーの読み取り専用で、「テンプレートで固定」と表示。
  - アイテム自選タグ：Toggle リスト。チェックすると自動的にそのタグのフィールドを追加、外すと自動的に削除。
- **属性（グループ表示、折りたたみ可能）**：

| グループ | ソース |
|----|------|
| テンプレート：XXX | テンプレート固有の属性フィールド |
| タグ名（テンプレート固定） | テンプレート固定タグの属性フィールド |
| タグ名 | アイテム自選タグの属性フィールド |
| その他（ソース削除済み） | ソーステンプレート / タグが削除された過去のフィールド（次回 Rebuild 時にクリア） |

同名フィールドが複数のソースから来る場合は競合警告が表示され、優先度（テンプレート固有 → テンプレート固定タグ → 自選タグ）で先着順に決まります。列挙フィールドである値を選択すると、その下にその列挙項目のサブ属性が読み取り専用で展開されます。

# 倉庫属性：重量とスタック

アイテム Inspector には、倉庫システムが使う 2 つのフィールドもあります：

| フィールド | 説明 |
|------|------|
| 重量 | 浮動小数点。0 = 重量なし（倉庫の重量上限計算に参加しない） |
| スタック上限 | 整数。0 = 無制限スタック、1 = スタック不可、>1 = 具体的な上限 |

# ランタイムクエリ

アイテムデータは静的定義で、ランタイムでは `InventoryDataManager` を通じてクエリします：

```csharp
using Ale.Inventory.Runtime;

// ID でアイテムを取得
Item item = InventoryDataManager.Instance.GetItem("sword_01");

// 属性値を取得（強く型付け + フォールバック）
int   atk  = item.GetAttributeValue<int>("攻撃力", 0);
string nm  = item.GetAttributeValue<string>("名称");

// 列挙表示名を取得
EnumType quality = InventoryDataManager.Instance.GetEnumType("品質");
string qName = quality?.GetItemByValue(item.GetAttributeValue<int>("品質"))?.name;

// タグを判定
bool isEquip = InventoryDataManager.Instance.ItemHasTag("sword_01", "装備");
```

属性読み取りの完全な API、表示文字列とソート換算は [属性システム](AttributeSystem_JA.md) を参照してください。

# アイテム関連 UI コンポーネント

`Runtime/UI/Item/` 以下のセルコンポーネントは、アイテムの静的データからアイコン / 名称 / 品質などを自動取得して表示します：

| コンポーネント | 説明 |
|------|------|
| `UiwInventoryItemSimple` | 簡易セル：アイコン + 数量のみ（通貨バーなど） |
| `UiwInventoryItemDetail` | 完全セル：アイコン / 名称 / 説明 / 品質背景 / 数量 / 価格 / ホバーハイライト / スタック満杯アニメーション |
| `UiwShopItemDetail` | ショップセル：品質背景 + アイコン + 名称 / 単価 + 残り取引可能回数 + 数量選択 |
| `UiwCraftingBlueprintCell` | ブループリントエントリ：主産出アイコン + ブループリント名 + ブループリント設定に応じた主産出アイテム属性の表示 |

各コンポーネントの Inspector パラメータとプレハブ作成は [UI コンポーネントガイド](UIComponentGuide_JA.md) を参照してください。

# よくある質問

**Q：列挙項目の表示順序を変えると、既存アイテムの列挙値は変わりますか？**
A：いいえ。列挙属性は不変の整数列挙値を保存しており、表示順序とは無関係です。

**Q：機能タグを削除するとアイテムのフィールドデータは失われますか？**
A：`RebuildAttributes` を呼ぶとソースが削除されたフィールドは除去されます。除去前は Inspector の「その他（ソース削除済み）」グループで確認できます。

**Q：Inspector に「列挙型が見つかりません」と表示される？**
A：列挙型名が改名されていないか確認してください。属性フィールド定義で列挙型を選び直せば修正できます。

**Q：エクスポートボタンがグレーアウトしている？**
A：空でない重複 ID を持つアイテム / 倉庫 / ショップ / ブループリントが存在します。まず赤くハイライトされたエントリを修正してください（空白 ID はエクスポート時に自動でスキップされ、ブロックしません）。
